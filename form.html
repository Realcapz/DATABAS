Documentation of this API

Routes

  • GET/users - returns all users.
  • GET/users/{id} - returns a user with the specified id or returns a 404 status if the user is not found.
  • POST/users - creates a new user accepting a JSON object in the format {"username": "", "password": "", "email": "", "name": ""}. username is mandatory; otherwise, a 422 message (unprocessable entity) is sent.
